Research

Sensor Input Spoofing in UAV Control Software

Current UAV control software (ArduPilot) is vulnerable to many different attacks in which an adversery is able to subtly influence the environment. Using SITL while running ArduPilot's source code, we were able to verify several of these vulnerabilities and developed attack models to demonstrate that a malicious party could disrupt, crash, or even gain some amount of control of a drone with the proper knowledge of ArduPilot and control of certain environmental factors. This flaw boils down to a lack of sensor fusion between sensors of different types. That is, sensors on the drone don't cross-validate input between orthagonal sensors if that input is acceptable, and as a result are vulnerable to environment-based attacks against certain sensors. We suggest new threat models be introduced that account for adverseries with this type of control and suggest software be developed that mitigate these types of attacks.
